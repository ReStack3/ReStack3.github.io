SQL

一、SQL 概述

1、什么是 SQL

SQL（Structured Query Language，结构化查询语言）是一种用于管理关系型数据库的编程语言。它用于执行查询、插入、更新、删除等数据库操作。SQL 是标准化的语言，可以用于与不同的数据库系统（如 MySQL、PostgreSQL、SQL Server、Oracle 等）交互。

2、语法要求

  • 大小写不敏感：SQL 关键字如 SELECT、FROM、WHERE 等不区分大小写，虽然通常建议使用大写来增强可读性。
  • 查询语句结束符：SQL 语句通常以分号 (;) 结束，尤其在需要执行多个语句时。
  • 空格与换行：空格和换行用来分隔SQL语句中的不同部分，增加可读性，SQL 引擎会忽略多余的空格和换行。
  • 字符串的引号：在 SQL 中，字符串常量使用单引号（'）括起来，例如 'Hello World'。
  • 表名和列名：可以用反引号（```）或双引号（"）括起来，避免与SQL关键字冲突。

二、SQL 分类

  • DDL（Data Definition Language）：数据定义语言，用来定义数据库对象：库、表、列等；

  • DML（Data Manipulation Language）：数据操作语言，用来定义数据库记录（数据）；

  • DCL（Data Control Language）：数据控制语言，用来定义访问权限和安全级别；

  • DQL（Data Query Language）：数据查询语言，用来查询记录（数据）

2.数据类型

MySQL 与 Java、C 一样，也有数据类型MySQL 中数据类型主要应用在列上。
常用类型：

数据类型 说明
int 整型，适用于存储整数。
double 浮点型，适用于存储带有小数的数字，例如 double(5,2) 表示最多 5 位数字，其中有 2 位小数。
decimal 精确的浮动小数类型，常用于存储财务数据等要求高精度的数值，不会丢失精度。
char 固定长度字符串类型。输入的字符串长度不够时会自动补充空格。
varchar 可变长度字符串类型，适用于存储长度不固定的字符串。
text 长文本类型，存储大量的字符数据。
blob 字节类型，通常用于存储二进制数据（如图片、音频等）。
date 日期类型，格式为 yyyy-MM-dd，用于存储日期值。
time 时间类型，格式为 hh:mm:ss，用于存储时间值。
timestamp 时间戳类型，存储从 1970 年 1 月 1 日以来的秒数，用于记录时间的变化。

四、DML（Data Manipulation Language）：数据操作语言

以下是插入数据、修改数据、删除数据的常见 SQL 语法，以及代码解读的表格：

操作 SQL 语句 说明
插入数据（指定列） sql INSERT INTO stu(sid, sname, age, gender) VALUES('s_1001', 'zhangSan', 23, 'male'); 向 stu 表插入一行数据，指定插入的列及其对应的值。
插入数据（指定部分列） sql INSERT INTO stu(sid, sname) VALUES('s_1001', 'zhangSan'); 向 stu 表插入一行数据，仅指定了部分列（sid 和 sname）。其他列会使用默认值或 NULL。
插入数据（不指定列） sql INSERT INTO stu VALUES('s_1002', 'liSi', 32, 'female'); 向 stu 表插入一行数据，按照表中列的顺序插入所有列的值。
修改数据 sql UPDATE stu SET sname='zhangSanSan', age=32, gender='female' WHERE sid='s_1001'; 更新 stu 表中 sid 为 s_1001 的记录，将 sname、age 和 gender 修改为新值。
修改数据（条件更新） sql UPDATE stu SET sname='liSi', age=20 WHERE age>50 AND gender='male'; 更新 stu 表中所有年龄大于 50 且性别为 male 的记录，将 sname 改为 liSi，age 改为 20。
修改数据（多条件更新） sql UPDATE stu SET sname='wangWu', age=30 WHERE age>60 OR gender='female'; 更新 stu 表中所有年龄大于 60 或性别为 female 的记录，将 sname 改为 wangWu，age 改为 30。
修改数据（NULL 检查） sql UPDATE stu SET gender='female' WHERE gender IS NULL; 将 stu 表中所有性别为空（NULL）的记录的 gender 字段更新为 female。
修改数据（递增更新） sql UPDATE stu SET age=age+1 WHERE sname='zhaoLiu'; 将 stu 表中所有 sname 为 zhaoLiu 的记录的 age 字段加 1。
删除数据（指定条件） sql DELETE FROM stu WHERE sid='s_1001'; 删除 stu 表中 sid 为 s_1001 的记录。
删除数据（多个条件） sql DELETE FROM stu WHERE sname='chenQi' OR age > 30; 删除 stu 表中 sname 为 chenQi 或年龄大于 30 的记录。
删除数据（删除所有记录） sql DELETE FROM stu; 删除 stu 表中的所有记录，但表结构不变，删除的记录可以回滚（事务）。
快速删除所有记录 sql TRUNCATE TABLE stu; 删除 stu 表中的所有记录，并且该操作无法回滚，效率比 DELETE 高。

五、DCL（Data Control Language）：数据控制语言

操作 SQL 语句 说明
创建用户 sql CREATE USER 'user1'@'localhost' IDENTIFIED BY '123'; 创建一个用户名为 user1，并且限制只能从 localhost 登录的用户，密码为 123。
sql CREATE USER 'user2'@'%' IDENTIFIED BY '123'; 创建一个用户名为 user2，允许从任何地址 (% 代表所有 IP 地址) 登录的用户，密码为 123。
给用户授权 sql GRANT CREATE, ALTER, DROP, INSERT, UPDATE, DELETE, SELECT ON mydb1.* TO 'user1'@'localhost'; 授予 user1 在 mydb1 数据库上的多种权限，包括 CREATE、ALTER、DROP、INSERT、UPDATE、DELETE、SELECT。
sql GRANT ALL ON mydb1.* TO 'user2'@'localhost'; 授予 user2 在 mydb1 数据库上的所有权限。
撤销授权 sql REVOKE CREATE, ALTER, DROP ON mydb1.* FROM 'user1'@'localhost'; 撤销 user1 在 mydb1 数据库上的 CREATE、ALTER、DROP 权限。
查看用户权限 sql SHOW GRANTS FOR 'user1'@'localhost'; 查看 user1 用户在 localhost 上的授权权限。
删除用户 sql DROP USER 'user1'@'localhost'; 删除 user1 用户。
修改用户密码（以root身份） sql USE mysql; 切换到 mysql 系统数据库，在该数据库下管理用户及权限。
sql ALTER USER 'user1'@'localhost' IDENTIFIED BY 'newpassword'; 修改 user1 用户的密码为 newpassword。

六、DQL（Data Query Language）：数据查询语言

数据库执行 DQL 语句不会对数据进行改变，而是让数据库发送结果集给客户端。

语法：
select 列名 —-> 要查询的列名称
from 表名 —-> 要查询的表名称
where 条件 —-> 行条件
group by 分组列 —-> 对结果分组
having 分组条件 —-> 分组后的行条件
order by 排序列 —-> 对结果分组
limit 起始行, 行数 —-> 结果限定

sql语言执行顺序

1. 查询解析

词法分析：将 SQL 查询字符串分解为 token，如 SELECT、name、AVG(salary)、FROM、employees 等。

语法分析：检查语法是否正确，生成解析树（Parse Tree）。

语义分析：

  • 检查 employees 和 departments 表是否存在。
  • 检查 dept_id、id、age、salary 等列是否存在。
  • 检查用户是否有权限访问这些表和列。

2. 查询优化

生成执行计划：

MySQL 优化器会考虑多种执行计划，例如：

  1. 先过滤 employees.age > 30，再与 departments 表连接。
  2. 先连接 employees 和 departments，再过滤 employees.age > 30。

优化器会根据统计信息（如表大小、索引分布等）选择成本最低的执行计划。

索引选择：

如果 employees.age 和 employees.dept_id 有索引，优化器可能会使用索引加速过滤和连接。如果 departments.id 有索引，优化器会使用索引加速连接操作。

连接顺序：

优化器决定先访问 employees 表还是 departments 表。通常，优化器会选择较小的表作为驱动表。

3. 查询执行

步骤 1：过滤 (WHERE employees.age > 30)

执行引擎扫描 employees 表，过滤出 age > 30 的行。如果 employees.age 有索引，使用索引快速定位符合条件的行。如果没有索引，执行全表扫描。

步骤 2：连接 (JOIN departments ON employees.dept_id = departments.id)

将过滤后的 employees 表与 departments 表连接。

如果 employees.dept_id 和 departments.id 有索引，使用索引加速连接。如果没有索引，执行嵌套循环连接（Nested Loop Join）或哈希连接（Hash Join）。

步骤 3：分组 (GROUP BY departments.name)

根据 departments.name 对连接后的结果集进行分组。

计算每个组的 AVG(salary)。如果结果集较大，可能会使用临时表存储分组结果。

步骤 4：过滤分组结果 (HAVING AVG(salary) > 50000)

对分组后的结果集进行过滤，只保留 AVG(salary) > 50000 的组。

步骤 5：排序 (ORDER BY AVG(salary) DESC)

对过滤后的分组结果按 AVG(salary) 降序排序。如果结果集较大，可能会使用文件排序（File Sort）。

步骤 6：限制结果集 (LIMIT 10)

只返回排序后的前 10 行。

4. 结果返回

将最终的结果集返回给客户端。结果集包含两列：name（部门名称）和 AVG(salary)（平均工资）。释放查询过程中使用的资源（如临时表、内存等）。

详细执行流程图

  1. **过滤 employees.age > 30**：

    • 输入：employees 表。
    • 输出：符合条件的 employees 行。

  2. **连接 employees 和 departments**：

    • 输入：过滤后的 employees 表和 departments 表。
    • 输出：连接后的结果集。

  3. **分组 GROUP BY departments.name**：

    • 输入：连接后的结果集。
    • 输出：按 departments.name 分组的结果集。

  4. **计算 AVG(salary)**：

    • 输入：分组后的结果集。
    • 输出：包含 AVG(salary) 的分组结果集。

  5. **过滤 HAVING AVG(salary) > 50000**：

    • 输入：分组结果集。
    • 输出：符合条件的分组结果集。

  6. **排序 ORDER BY AVG(salary) DESC**：

    • 输入：过滤后的分组结果集。
    • 输出：按 AVG(salary) 降序排序的结果集。

  7. **限制 LIMIT 10**：

    • 输入：排序后的结果集。
    • 输出：最终返回的 10 行结果。
