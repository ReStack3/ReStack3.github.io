整体流程

阶段 步骤 操作 目的
1. 源程序 APK
1.1 准备原始 APK 获取原始 APK 文件 从开发环境中导出未加密的 APK 文件。 提供需要保护的原始应用程序。
1.2 提取 DEX 文件 解压 APK，提取 classes.dex 使用工具（如 apktool）解压 APK 文件，获取原始 DEX 文件。 为加密和加壳提供原始 DEX 文件。
2. 加壳程序
2.1 选择加密算法 选择加密算法（如 AES） 使用对称加密算法对原始 DEX 文件进行加密。 保护原始 DEX 文件的内容，防止反编译和篡改。
2.2 加密 DEX 文件 编写加密工具，加密 DEX 文件 将原始 DEX 文件加密为二进制数据。 生成加密后的 DEX 文件，供后续与壳程序合并。
2.3 编写壳程序 编写壳程序 壳程序包含解密逻辑和 DEX 文件加载逻辑（如通过 DexClassLoader 加载）。 壳程序负责在运行时解密 DEX 文件并加载到内存中执行。
2.4 修改 DEX 文件头 更新 DEX 文件头 计算并更新 checksum、signature 和 file_size 字段。 确保 DEX 文件的完整性和正确性。
2.5 合并壳程序与加密 DEX 文件 将加密 DEX 文件嵌入壳程序 将加密后的 DEX 文件作为资源或二进制数据嵌入壳程序的 APK 中。 将壳程序和加密 DEX 文件打包为一个整体，便于分发和运行。
2.6 重新打包 APK 打包 APK 将修改后的 DEX 文件、壳程序和其他资源打包为 APK 文件。 生成最终的加壳 APK 文件，供安装和运行。
2.7 签名 APK 对 APK 进行签名 使用 Android 签名工具（如 apksigner）对 APK 文件进行签名。 确保 APK 文件的完整性和来源可信性。
2.8 对齐优化 对齐优化 APK 使用 zipalign 工具对 APK 文件进行对齐优化。 提高 APK 文件的加载和运行效率。
3. 脱壳程序
3.1 运行壳程序 启动加壳 APK 用户安装并运行加壳 APK，壳程序开始执行。 壳程序负责解密和加载加密的 DEX 文件。
3.2 解密 DEX 文件 壳程序解密 DEX 文件 壳程序从资源中提取加密 DEX 文件并解密。 将加密的 DEX 文件还原为原始 DEX 文件。
3.3 加载 DEX 文件 加载解密后的 DEX 文件 使用 DexClassLoader 或其他方式将解密后的 DEX 文件加载到内存中。 使解密后的 DEX 文件能够正常执行。
3.4 执行原始逻辑 运行原始 DEX 文件 解密后的 DEX 文件被加载并执行原始应用程序逻辑。 完成应用程序的正常运行。

其实Application即使不创建，系统也会帮我们创建的。但是创建自定义Application类是为了方便反射获取Application类，进而创建源程序的Application实例来替换原有的壳程序的Application实例。
