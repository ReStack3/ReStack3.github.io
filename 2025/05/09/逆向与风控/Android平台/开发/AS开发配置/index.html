简单介绍

Gradle 是一款开源自动化构建工具，其设计哲学围绕灵活性和高性能展开，被广泛应用于 Java、Kotlin、Android、C++ 等多语言项目的全生命周期管理。

其作用类似于 Makefile ,旨在通过预定义规则将源代码转换为可执行程序或部署包，解决手动编译的低效问题。

由于Android Studio已经集成了Gradle,我们并不需要事无巨细的进行构建，仅仅知道一些常见的命令和文件即可。

Gradle目录结构

接下来结合一个Android项目来整体分析一下：

Build.gradle

根目录下的build.gradle通常是顶层的配置，包括整个项目的仓库地址和Gradle插件版本，而app目录下的则处理具体模块的构建配置，比如依赖项、编译版本等。 还需要考虑Gradle的多项目结构，即一个项目可以包含多个模块，每个模块有自己的build.gradle。因此，根目录的build.gradle负责全局设置，而模块级的处理各自特定的配置。

1-【APP目录下】

模块目录 app/build.gradle

  • 定位：模块级“地方法规”，定义当前模块专属行为
  • 核心职责：
    • 应用插件（如com.android.application 或quantum.kotlin ）
    • 模块特化配置（如compileSdk 35、minSdk 28）
    • 依赖项声明（如本地库）
3-【gradle目录下】

根目录 build.gradle

  • 定位：项目级“宪法”，定义全模块共享的规则
  • 典型配置项：
    • 构建工具链版本
    • 全局仓库镜像（如阿里云、Google量子仓库）
    • 跨模块依赖管理策略（如统一依赖版本约束）

gradle-wrapper.properties

gradle.properties

作用：定义项目级全局属性和构建优化策略

local.properties

作用：声明开发者本地环境专属配置,比如：SDK 路径

settings.gradle

作用：定义项目结构与模块关系

SDK

Android SDK（Software Development Kit）是用于开发Android应用程序的工具集合，包含了开发、测试和调试应用所需的库、工具和文档。

Android SDK 的主要组件

  1. API 库：提供访问Android系统功能的接口。

  2. 开发工具：如ADB（Android Debug Bridge）、Fastboot等，用于调试和管理设备。

    • adb.exe（Android Debug Bridge）：用于连接安卓设备与PC端，进行调试和控制设备操作

    • aapt.exe：用于将开发的应用打包成APK安装文件

    • dx.bat：将.class字节码文件转成安卓字节码.dex文件

  3. 系统镜像：用于在模拟器上测试应用。

为什么 Android Studio 需要 SDK

  1. 编译和构建：Android Studio 依赖 SDK 中的工具和库来编译代码并生成APK文件。
  2. 调试：SDK 提供调试工具（如ADB）帮助开发者排查问题。
  3. 模拟器支持：SDK 提供系统镜像，用于在模拟器上运行和测试应用。
  4. API 支持：SDK 包含不同版本的API库，确保应用兼容不同Android版本。

SDK 目录结构

build-tools

  • 包含用于构建Android应用的命令行工具，如 aapt、dx、zipalign 等。
  • 每个Android版本可能有不同的工具版本。

platforms

  • 包含已安装的Android平台版本（如Android 10、Android 11等）。
  • 每个平台目录下有对应的Android系统库（如 android.jar）和资源文件。

platform-tools

  • 包含与平台相关的工具，如 adb（Android Debug Bridge）、fastboot 等。
  • 这些工具用于设备调试、刷机等操作。

tools

  • 包含通用的开发工具，如 emulator（模拟器）、sdkmanager（SDK管理工具）、avdmanager（虚拟设备管理工具）等。

system-images

  • 包含用于Android模拟器的系统镜像文件。
  • 每个镜像对应特定的Android版本和CPU架构（如ARM、x86）。

sources

  • 包含Android平台的源代码，方便开发者查看API的实现细节。

docs

  • 包含Android开发的官方文档，提供API参考和开发指南。

extras

  • 包含额外的工具和库，如Google Play服务、支持库、硬件驱动等。

**ndk**（如果安装了NDK）

  • 包含Android Native Development Kit（NDK），用于开发C/C++原生代码。

emulator

  • 包含Android模拟器的相关文件和工具。

patcher

  • 用于SDK的更新和补丁管理。

licenses

  • 包含SDK及其组件的许可协议文件。

自然对应着sdk中的Android模拟器的系统镜像文件。当然APP也可以用USB连接到真机上运行。也有人会问：为什么有时候也可以进行配置在雷神模拟器上运行？
