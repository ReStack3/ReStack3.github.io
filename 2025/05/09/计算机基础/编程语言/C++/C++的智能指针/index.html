C++的智能指针

在C++11特性 - The Peak Tower文章中简答提到了“资源管理指针”这一个概念，却没有深入的去了解，本文将重点看看所谓智能指针到底是什么？

RAll思想

RAII（Resource Acquisition Is Initialization，资源获取即初始化） 是 C++ 中的一种核心编程思想，它的核心是通过对象的生命周期来管理资源。RAII 是智能指针（如 std::unique_ptr 和 std::shared_ptr）的指导思想，也是 C++ 中管理资源（如内存、文件句柄、网络连接、互斥量等）的最佳实践。

RAII 的核心思想

  1. 资源获取即初始化：
    • 在对象的构造函数中获取资源（例如分配内存、打开文件、加锁等）。
    • 在对象的析构函数中释放资源（例如释放内存、关闭文件、解锁等）。
  2. 利用对象的生命周期管理资源：
    • 当对象创建时，资源被获取。
    • 当对象销毁时，资源被释放。
    • 这种方式确保了资源的正确释放，即使发生异常也不会泄漏资源。

RAII 的设计原则

  1. 封装资源管理：
    • 将资源的获取和释放封装在类的构造函数和析构函数中。
  2. 确保析构函数释放资源：
    • 析构函数必须正确释放资源，即使发生异常。
  3. 禁止拷贝（如果需要）：
    • 如果资源不能被共享，可以通过删除拷贝构造函数和拷贝赋值运算符来禁止拷贝。

unique_ptr

std::unique_ptr 是 C++11 引入的一种智能指针，用于管理动态分配的内存资源。它提供独占式所有权（exclusive ownership），确保一个时刻只有一个 unique_ptr 拥有所管理的对象，并在指针超出作用域时自动释放内存。

基本特性

  • 独占所有权：一个 unique_ptr 独占所管理的对象，不能复制，只能移动。
  • 自动释放：当 unique_ptr 超出作用域时，会自动释放所管理的对象。
  • 轻量高效：unique_ptr 的开销几乎与原始指针相同，没有额外的引用计数。

基本用法

创建 unique_ptr

  • **std::make_unique**：C++14 引入的工厂函数，用于创建 unique_ptr，更安全且更易读。

释放所有权

  • **release()**：释放 unique_ptr 对对象的所有权，返回原始指针，unique_ptr 变为 nullptr。

重置指针

  • **reset()**：释放当前管理的对象（如果有），并接管新对象。

检查是否为空

  • **operator bool()**：检查 unique_ptr 是否为空。

独占所有权

unique_ptr 不能复制，只能移动。这是其独占所有权的核心特性。

移动语义

  • **std::move()**：用于转移 unique_ptr 的所有权。

禁止复制

  • 禁止复制：unique_ptr 的拷贝构造函数和拷贝赋值运算符被删除。

shared_ptr

深入理解Modern C++智能指针std::shared_ptr - 知乎

C++智能指针shared_ptr是一种可以自动管理内存的智能指针，它是C++11新增的特性之一。与传统指针不同，shared_ptr可以自动释放所管理的动态分配对象的内存，并避免了手动释放内存的繁琐操作，从而减少了内存泄漏和野指针的出现。

看百度来的解释，结合最近在看各种语言的垃圾回收机制(GC)，个人觉得该所谓的智能指针只是对C++没有垃圾回收机制的一种弥补。比如python的GC机制也有引用计数类似的概念。

那么重点是shared_ptr是如何实现自动释放所管理的动态分配对象的内存的呢？

shared_ptr是一个模板类，通过引用计数器实现多个智能指针共享对一个对象的所有权。每次复制一个shared_ptr对象时，该对象的引用计数器会增加1，当最后一个shared_ptr对象被销毁时，引用计数器减1，如果引用计数器变为0，则释放所管理的对象的内存。

C++ 的智能指针（如 std::shared_ptr、std::unique_ptr 和 std::weak_ptr）是类模板，它们封装了原始指针，并提供了额外的功能（如自动内存管理、引用计数等）。智能指针的实例是一个对象，但它通过重载运算符（如 operator* 和 operator->）表现得像一个指针。(shared_ptr 代表的是共享所有权，即多个 shared_ptr 可以共享同一块内存。)

内存泄露(循环引用)

在对象A中通过shared_ptr管理资源B，在对象B中通过shared_ptr管理资源A。导致A，B均不会释放相应内存，产生内存泄露。

性能差异

注意：shared_ptr 的内存占用是裸指针的两倍。因为除了要管理一个裸指针外，还要维护一个引用计数。因此相比于 unique_ptr, shared_ptr 的内存占用更高

考虑到线程安全问题，引用计数的增减必须是原子操作。而原子操作一般情况下都比非原子操作慢。shared_ptr 在性能上固然是低于 unique_ptr。而通常情况，我们也可以尽量避免 shared_ptr 复制。

如果，一个 shared_ptr 需要将所有权共享给另外一个新的 shared_ptr，而我们确定在之后的代码中都不再使用这个 shared_ptr，那么这是一个非常鲜明的移动语义。对于此种场景，我们尽量使用 std::move，将 shared_ptr 转移给新的对象。因为移动不用增加引用计数，性能比复制更好。

weak_ptr

std::weak_ptr 是 C++11 引入的智能指针类型，用于解决 std::shared_ptr 的循环引用问题。它提供了一种不参与对象引用计数的弱引用，这意味着它不会影响所管理对象的生命周期。

weak_ptr是为了配合shared_ptr而引入的一种智能指针，它指向一个由shared_ptr管理的对象而不影响所指对象的生命周期。(也就是将一个weak_ptr绑定到一个shared_ptr不会改变shared_ptr的引用计数。)

它只可以从一个 shared_ptr 或另一个 weak_ptr 对象构造，它的构造和析构不会引起引用记数的增加或减少。弱引用能检测到所管理的对象是否已经被释放，从而避免访问非法内存。

std::weak_ptr没有重载 \* 和 -> 运算符因为其不能直接访问资源，因为它不拥有资源的所有权。

也就是说，weak_ptr是为了弥补shared_ptr循环引用而生的，它没有RAII的特性，不直接管理资源，只是shared_ptr的跟班，这也是weak_ptr支持使用shared_ptr构造的原因。

在使用上，weak_ptr支持指针所有的操作。它不是一个功能型的智能指针，而是辅助型，它的使命是解决shared_ptr造成的循环引用问题。

强弱引用

在C++中，强弱引用是用来描述对象生命周期管理的概念，通常与智能指针（如std::shared_ptr和std::weak_ptr）相关。它们的主要区别在于是否影响对象的生命周期。

强引用（Strong Reference）

强引用是指对对象的引用会增加对象的引用计数，从而延长对象的生命周期。只要存在至少一个强引用，对象就不会被销毁。

特点

  • 影响生命周期：强引用会增加对象的引用计数，防止对象被销毁。
  • 可能导致循环引用：如果两个对象相互持有对方的强引用，会导致引用计数无法归零，从而引发内存泄漏。

在C++中，std::shared_ptr是一种强引用智能指针。它通过引用计数来管理对象的生命周期。

弱引用（Weak Reference）

弱引用是指对对象的引用不会增加对象的引用计数，因此不会影响对象的生命周期。弱引用通常用于观察对象是否存在，而不会阻止对象被销毁。

特点

  • 不影响生命周期：弱引用不会增加对象的引用计数，不会阻止对象被销毁。
  • 解决循环引用问题：通过将循环引用中的一方改为弱引用，可以避免内存泄漏。
  • 需要检查对象是否存在：在使用弱引用访问对象时，需要检查对象是否仍然存在。

在C++中，std::weak_ptr是一种弱引用智能指针。它不会增加对象的引用计数，但可以通过lock()方法获取一个临时的std::shared_ptr来访问对象。
