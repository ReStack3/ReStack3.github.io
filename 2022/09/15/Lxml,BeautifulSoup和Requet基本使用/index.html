Lxml和BeautifulSoup基本使用

Lxml

lxml - Processing XML and HTML with Python

lxml 是一个用于处理 XML 和 HTML 的 Python 库，它提供了对 XML 和 HTML 的高效解析和操作能力。

HTML 和 XML 之间的区别

XML 是一种面向数据的标记语言，专注于描述和传输结构化信息；它注重可扩展性和自定义，适合数据存储和交换。

HTML 是一种面向显示的标记语言，专注于内容的呈现和用户交互；它结构固定且宽松，适合网页设计。

特性 XML（可扩展标记语言） HTML（超文本标记语言）
目的 用于表示、存储和传输数据，强调数据的结构化和自定义。 用于展示网页内容，描述文档的外观和行为。
语法规则 严格：标签必须正确嵌套，属性值必须用引号，必须有根标签。 宽松：标签可以不完全闭合，不区分大小写，错误不会阻止文档渲染。
标签定义 用户可以自定义标签。 使用预定义的固定标签（如 <html>、<div>）。
数据格式 数据驱动：描述数据及其结构，通常无视觉呈现。 视觉驱动：用来定义文档的结构和视觉内容。
用途 数据交换（如配置文件、Web 服务接口）。 构建和展示网页内容。
可扩展性 完全支持：用户可以根据需求定义自己的标签和结构。 不支持扩展：只能使用 HTML 标准定义的标签。
显示功能 仅存储数据，无法直接定义数据如何显示。 专为显示内容设计，提供文本、图像、超链接等显示功能。
严格性 严格遵守规则，错误会导致解析失败。 宽容性强，小错误不会影响页面渲染。
案例用途 配置文件（如 .xml 文件）、数据交换（如 SOAP、RSS）。 网页设计、超媒体文档展示。
属性 属性仅用于提供额外信息，内容以文本为主。 属性用于控制页面的外观和行为（如 style、class）。
大小写敏感性 区分大小写：<Tag> 和 <tag> 是不同的标签。 不区分大小写：<Tag> 和 <tag> 是等价的。
解析工具 通常需要专用的解析器（如 lxml、ElementTree）。 浏览器原生支持，无需额外解析工具。
支持的标准 W3C XML 标准。 W3C HTML 和 HTML5 标准。
数据验证 支持数据验证（如通过 DTD 或 XML Schema 验证）。 不支持数据验证，仅为文档展示服务。
与 CSS 的集成 通常不直接集成 CSS，需结合 HTML 或其他技术实现样式。 天生与 CSS 集成，用于控制页面样式和布局。
与 JavaScript 的集成 通常作为数据源（如通过 AJAX 获取和解析 XML 数据）。 可以直接与 JavaScript 结合，实现交互式网页功能。
人类可读性 设计为机器和人类都易于理解，适合存储复杂的结构化数据。 面向人类设计，主要用于网页内容和交互的编写。

lxml 模块组成及功能概览

模块 功能描述 常见方法/功能
lxml.etree 核心模块，用于解析、生成、查询和修改 XML 和 HTML 文档。 parse()、fromstring()、Element()、XPath()、tostring()、节点操作（如 append()、remove()）。
lxml.html 专门处理 HTML，支持修复不规范 HTML 结构并提供更高级操作接口。 html.parse()、html.fromstring()、html.tostring()、CSS 选择器功能（配合 CSSSelector）。
lxml.xpath 提供 XPath 查询功能，用于快速查找和筛选 XML/HTML 元素。 XPath()、XPathResult()，通过 element.xpath() 执行查询。
lxml.objectify 将 XML 数据转换为 Python 对象，支持属性访问方式操作 XML 数据。 objectify.fromstring()、objectify.Element()，动态生成属性和子元素。
lxml.etree.CDATA 用于表示 XML 文档中的 CDATA 部分，处理不需要转义的字符数据。 无直接方法，适用于定义和操作 CDATA 节点内容。
lxml.html.CSSSelector 提供 CSS 选择器支持，通过 CSS 风格选择器查找文档中的元素（类似 jQuery）。 CSSSelector(selector)，基于 CSS 选择器的元素查询。
lxml.etree.XMLParser XML 解析器，支持标准的 XML 文档解析。 XMLParser()，支持自定义解析行为。
lxml.etree.HTMLParser HTML 解析器，支持修复不规范 HTML 并解析为 HTML 文档。 HTMLParser()，适合处理不规范的网页内容。
lxml.builder 提供动态构建 XML 文档的能力，通过类对象生成 XML 元素并组装完整文档。 builder.ElementMaker()，创建元素构建器，支持动态 XML 生成。

模块功能对比与适用场景

模块名称 适用场景
lxml.etree 通用 XML 和 HTML 操作，适用于标准文档的解析、修改和生成。
lxml.html 处理 HTML 文档的高级需求，适合网页爬虫和修复不规范 HTML。
lxml.xpath 基于 XPath 的快速节点查询，适合大规模 XML/HTML 数据的精准查找。
lxml.objectify 将 XML 转为对象，适合需要通过属性访问 XML 节点的场景。
lxml.builder 动态生成 XML，适用于需要灵活构建复杂 XML 文档的情况。
lxml.html.CSSSelector 使用 CSS 选择器查找 HTML 元素，适用于熟悉 CSS 的开发者操作 HTML。
XMLParser/HTMLParser 定制 XML/HTML 解析行为，适合需要特殊解析规则的应用。

lxml.etree 和 lxml.html 对比

对比维度 lxml.etree lxml.html
模块性质 核心模块，用于解析、生成、查询和修改 XML 和 HTML。 基于 lxml.etree 的扩展模块，专门处理 HTML 文档，特别是不规范 HTML。
支持文档类型 XML 和 HTML，主要面向标准化文档，要求严格符合 XML 规范。 专注 HTML，适用于不规范 HTML 文档，自动修复结构问题。
解析器 标准的 XML 解析器（XMLParser 和 HTMLParser），对标准要求严格。 使用基于 HTML 的解析器，对缺失闭合标签、不匹配标签等容错处理更强。
容错能力 面向 XML 标准，容错能力较弱。 面向 HTML 标准，自动修复常见的 HTML 错误，例如闭合标签补全、修复嵌套问题。
功能扩展 提供标准的 XML/HTML 解析和操作功能，支持 XPath、XSLT、节点增删改查等操作。 在继承 lxml.etree 功能的基础上，提供 CSS 选择器支持、HTML 特定修复功能以及对不规范文档的优化。
自动修复 仅对 XML 规范严格处理，不会自动修复文档结构问题。 能修复不规范的 HTML，例如缺失的 </html> 或不完整的标签。
CSS 选择器支持 不支持 CSS 选择器，只能通过 XPath 查询。 支持 CSS 选择器查询，提供 CSSSelector 类，简化 HTML 元素的查询。
适用场景 需要严格处理 XML 或标准 HTML 的场景，如配置文件、API 数据解析等。 适合处理网页爬虫和 HTML 文档操作，尤其是不规范 HTML 的解析和修复场景。
示例方法 etree.parse()、etree.fromstring()、etree.Element()、XPath()。 html.parse()、html.fromstring()、CSSSelector()、自动补全和修复 HTML 结构问题。

lxml.etree 常用属性与方法

1. 核心类及作用

类名 作用
Element 表示 XML/HTML 文档中的节点，支持各种节点操作。
ElementTree 表示整个文档树结构，用于管理和操作根节点及其子节点。
XMLParser 用于解析 XML 文档，支持自定义解析行为。
HTMLParser 用于解析 HTML 文档，支持容错处理。
XPath 表示 XPath 表达式，便于对文档进行高级查询。
XSLT 表示 XSLT 转换对象，用于对 XML 文档进行样式转换。

2. Element 类常用属性和方法

属性/方法 说明
属性
tag 返回节点的标签名（字符串）。
attrib 返回节点的属性字典（键值对可修改）。
text 返回节点的文本内容（可读写）。
tail 返回节点的尾部文本（紧接在节点后面的文本）。
方法
append(element) 向当前节点的子节点列表中追加一个子元素。
extend(elements) 向当前节点追加多个子元素。
find(path) 按 XPath 查询子节点，返回第一个匹配的子节点。
findall(path) 按 XPath 查询所有匹配的子节点，返回列表。
findtext(path) 按 XPath 查询子节点并返回其文本内容。
get(key, default=None) 获取指定属性的值，若不存在返回默认值。
set(key, value) 设置指定属性的值。
keys() 返回节点所有属性的键。
items() 返回节点所有属性的键值对。
insert(index, element) 在指定位置插入子节点。
remove(element) 删除指定的子节点。
clear() 清除当前节点的所有内容（文本、子节点和属性）。
iter(tag=None) 深度优先遍历当前节点及其所有子节点。
iterchildren(tag=None) 遍历当前节点的直接子节点。
itertext() 遍历当前节点及其子节点中的所有文本内容。

XPath 的基本语法规则

lxml 库的 XPath 语法与使用指南

XPath（XML Path Language）是一种用于定位 XML/HTML 文档中节点的语言，它支持节点查询、条件筛选以及多种操作功能。在 Python 的 lxml.etree 中，XPath 是 XML 和 HTML 查询的核心。

以下内容将介绍 XPath 的常用语法规则和 lxml 中的使用方法。

1. XPath 的基本语法规则

1.1 基础节点选择

表达式 说明 示例
/ 根节点选择。 /root 选择根节点 <root>。
// 从文档的任意位置选择匹配的节点（不局限于直接子节点）。 //child 选择文档中所有的 <child> 节点。
. 当前节点。 . 表示当前节点。
.. 父节点。 .. 表示当前节点的父节点。
@ 属性选择器，用于选择节点属性值。 //@id 选择所有节点的 id 属性。

1.2 条件筛选

表达式 说明 示例
[condition] 筛选符合条件的节点。 //child[@name='A'] 选择属性 name="A" 的 <child> 节点。
position() 返回节点的当前位置（从 1 开始）。 //child[position()=1] 选择第一个 <child> 节点。
last() 返回节点集的最后一个位置。 //child[last()] 选择最后一个 <child> 节点。

1.3 文本选择

表达式 说明 示例
text() 选择节点的文本内容。 //child/text() 选择 <child> 节点的文本内容。
contains() 检查字符串是否包含子串。 //child[contains(@name, 'part')] 选择 name 属性包含 part 的节点。
starts-with() 检查字符串是否以某子串开头。 //child[starts-with(@name, 'prefix')] 选择 name 属性以 prefix 开头的节点。

1.4 逻辑与算术运算

运算符 说明 示例
and 与运算。 //child[@id='1' and @name='A']
or 或运算。 //child[@id='1' or @name='A']
= 等于。 //child[@id='1']
!= 不等于。 //child[@id!='1']
< / <= 小于 / 小于等于。 //child[@count<5]
> / >= 大于 / 大于等于。 //child[@count>10]
